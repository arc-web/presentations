Request lifecycle

How a URL becomes an action

1
MCP tool call arrives - Claude Code calls browser_navigate or browser_evaluate with a URL and session name
2
Router classifies domain - site_registry.json determines risk level, browser mode, and hourly rate limit
3
Rate limiter checks - Rolling 1-hour window per domain. Rejected cleanly if over limit
4
Session pool resolves - Existing session returned if alive. New Chrome launched if needed. Stale locks auto-cleaned
5
Stealth execution - Bezier cursor movement, log-normal delays, per-character typing. Indistinguishable from a human
